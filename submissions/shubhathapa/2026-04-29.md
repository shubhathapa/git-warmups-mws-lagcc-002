## Questions to answer in your daily log

- What did `reword` do vs what did `squash` do?
reword changes only the commit message without changing the content of the commit.
squash combines multiple commits into one commit and lets you write a new combined message. 

- Why might you rewrite history before pushing but not after?
to clean up mistakes and organize commits. After pushing, rewriting history can cause problems for others who already pulled the commits. 

- What does `pick` mean? What happens if you delete a `pick` line?
pick keeps the commit as it is. and if you delete the pick line, that commit is removed from the history during thr rebase. 



## Questions to answer in your daily log

- What's the difference between `fetch` and `pull`?
fetch downloads changes from the remote repository but does not merge them into the current branch. 
pull does both, it fetches the change and immediately merges them into my current branch

- What's the difference between `origin` and `upstream` when a repo has both?
origin is the personal fork or the main remote copy that I push to 
upstream is the original repository I forked from. 

- What does `-u` do on `git push -u origin <branch>`?
-u sets the upstream tracking relationship between my local branch and the remote branch . after you use it once, you can run only git push or git pull without typing the entire branch name.
